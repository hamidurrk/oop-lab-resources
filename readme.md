## Repository Structure

- **weekX/**: Each week has its own folder (e.g., `week1/`, `week2/`, ...).
    - **taskY/**: Each week folder contains multiple task folders (e.g., `task1/`, `task2/`, ...).
        - **Source Files**: Task folders contain starter code, instructions, or templates (e.g., `Tone.java`).

Example:
```
week2/
    task1/
    task2/
        Tone.java
```

---

## For Students
- Access the repository using the provided link or QR code.
- Navigate to the folder for the current week (e.g., `week2`).
- Open the relevant task folder and start working on the provided files.
- Follow provided instructions or templates.
- Save your work in the appropriate location as instructed by your TA.
- Ask your TA if you have any questions or need clarification.

---

## Contribution Guidelines
- **Students** should not push changes to this repository unless explicitly instructed.
- For issues or suggestions, contact your TA or course instructor.

## Best Practices
- Keep your workspace organized by working within the correct week and task folders.
- Do not modify files outside your assigned tasks.
- Read all instructions carefully before starting each lab.
